VoiceSDK

VoiceSDK is an SDK facade object. The inception point of all communications between client and SDK.

Properties

  • state

    The state refects the current SDK state. The client may adopt VoiceSDKObserverProtocol and subscribe for changes.

  • configuration

    The configuration contains SDK settings. It should be set at least once before activation.

  • authenticationContext

    The authenticationContext contains authentication token. It should be set at least once before activation.

  • networkReachabilityStatus

    The networkReachabilityStatus refelects the current network reachability status. The client may adopt NetworkReachabilityObserverProtocol and subscribe for changes.

  • calls

    The calls contains active calls. The client may adopt CallSetObserverProtocol and subscribe for changes.

  • pushToken

    The pushToken contains push notification token (PushKit). The client may adopt PushNotificationObserverProtocol and subscribe for changes.

Initializers

  • preinit()

    The preinit() function prepares the SDK for receiving VoIP push notifications.

    Note

    The application should call this function asap in application(_:didFinishLaunchingWithOptions:). But even so, it is recommended to setup logMessageCallback in the first place.
  • shared

    The shared is a reference on a VoiceSDK singleton (SDK facade object).

Functions

  • activate(completion:)

    The activate(completion:) function creates (setup) the internal SDK components. It should be called by app on user login (autologin) completion.

  • deactivate(completion:)

    The deactivate(completion:) function destroys the internal SDK components. It should be called by app upon user logout.

  • placeCall(with:completion:)

    The placeCall(with:completion) function places outgoing call.

    Precondition

    The VoiceSDK object should be in active state.
  • updatePhoneNumber(_:completion:)

    The updatePhoneNumber(:) function updates phone number.

  • updateInboundCallPath(_:completion:)

    The updateInboundCallPath(:) function updates inbound call path.

  • version()

    The version() function returns Voice SDK version.

  • voiceStackVersion()

    The voiceStackVersion() function returns version of Voice Stack used by Voice SDK.

Callbacks

  • logMessageCallback

    The logMessageCallback is used to route SDK log messages to client.

  • contactResolverCallback

    The contactResolverCallback is used to request Contact Info from client.

Notifications

  • addObserver(_:)

    The addObserver(_:) function adds observer object.

    Note

    The weak reference is stored.

    Precondition

    In order to be notified the observer should adopt at least one the following protocols:

    • VoiceSDKObserverProtocol

    • CallSetObserverProtocol

    • NetworkReachabilityObserverProtocol

    • PushNotificationObserverProtocol

  • removeObserver(_:)

    The removeObserver(_:) function removes observer object.
